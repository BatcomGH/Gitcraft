<div>
<p>
¿Quieres jugar Minecraft con tus amigos pero no tienen un servidor? <br>
¡Eso ya no será nunca más un problema!
</p>
</div>

<div>
<p>
Gitcraft es un proyecto que nace del problema <b>"quiero jugar con mis amigos en LAN pero quiero que ellos puedan abrir el mundo sin la necesidad de que esté presente"</b>.
</p>
</div>

<div>
<p>
Requisitos:
</p>
<ul>
<li>Git.</li>
<li>Una VPN: puede ser cualquiera que permita crear LANs virtuales, por ejemplo, Hamachi o Radmin.</li>
<li>Saber lo básico de git (por el momento esto es un poco requisito, en el futuro veré como evitarlo).</li>
</ul>
</div>

<div>
<p>
<b>¿Cómo usarlo?</b>
</p>
<p>Para crear tu mundo:</p>
<ol>
<li>Crear un repositorio remoto en tu plataforma de preferencia, como Github o Gitlab.</li>
<li>Ejecutar el .jar en consola con el comando "java -jar <i>nombredeljar</i>.jar".</li>
<li>Seguir los pasos que indica el programa y ¡listo!</li>
</ol>
<p>Una vez que el mundo esté en un repositorio:</p>
<ul>
<li>Solo sigan las instrucciones de la aplicación.</li>
<li>Convertir en colaboradores del repositorio remoto a los hosts. Si no se hace, no podrán subir sus cambios.</li>
<li>No es necesario que todos los jugadores clonen el repositorio, solo aquellos que pretendan ser hosts.</li>
<li>Cuando el host actual deje de jugar, siempre deberá que subir los cambios para que el siguiente host pueda descargarlos.</li>
</ul>
<p>Recomendaciones:</p>
<ul>
<li>Antes de abrir el mundo, siempre asegurarse de haber descargado los cambios del repositorio remoto.</li>
<li>Evitar entrar al mundo cuando no se es host.</li>
<li>Evitar usar el SSH para clonar o crear repositorios. Requiere de configuraciones avanzadas de Git.</li>
</ul>
</div>

<div>
<p>Por el momento, el proyecto solo se ha testeado en Windows. Posiblemente funcione en Linux, pero definitivamente no hay soporte para Mac (quizás pronto lo haya).</p>
</div>
